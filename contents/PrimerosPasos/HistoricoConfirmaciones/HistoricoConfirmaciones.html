Histórico de confirmaciones

Después de haber hecho varias confirmaciones, o si has clonado un repositorio que ya tenía un histórico de confirmaciones, probablemente quieras mirar atrás para ver qué modificaciones se han llevado a cabo. La herramienta más básica y potente para hacer esto es el comando git log, que al ser ejecutado orroja una salida similar a esta:

commit 230a1fbffceedfc7a4bf6320a1ac037c77acd5bc

Date:   Wed Aug 6 10:20:19 2014 -0500

    Seccion 2.2 completa

commit 8bc44e2575a64ec8c1f1cbcbca4ff80c3a01e615
Author: Jorge Luis <jorgeluisrmx@gmail.com>
Date:   Wed Aug 6 10:06:06 2014 -0500

    Eliminacion con git rm filename de la copia de seguridad de git.html

Como puedes ver, este comando lista cada confirmación con su suma de comprobación SHA-1, el nombre y dirección de correo del autor, la fecha y el mensaje de confirmación.

El comando git log proporciona gran cantidad de opciones para mostrarte exactamente lo que buscas. Aquí veremos algunas de las más usadas.

Limitando la salida del histórico

Git cuenta con una serie de opciones para limitar su salida —es decir, opciones que te permiten mostrar únicamente parte de las confirmaciones—.

mostrar las últimas \(n\) confirmaciones

Para mostrar los último \(n\) commits, agrega la opción -n donde \(n\in \{1,2,3,\ldots\}\):

git log -n

limitando por fecha

Es posible limitar las salidas del log por medio de las opciones --since y --after muestra aquellas confirmaciones hechas después de la fecha especificada. Mientras que --until y --before muestran aquellas confirmaciones hechas antes de la fecha especificada.

Por ejemplo, este comando lista todas las confirmaciones hechas durante las dos últimas semanas:

git log --since=2.weeks

Este comando acepta muchos formatos. Puedes indicar una fecha concreta (“2008-01-15”), o relativa, como "2 years 1 day 3 minutes ago".

filtrando por autor

La opción --author te permite filtrar por autor:

git log --author=gitster

filtrando por palabras clave

Para filtrar usando palabras claves en los mensajes de confirmación usa la opción --grep.

Por otro lado si deseas excluir todas las confirmaciones correspondientes a uniones (merges) usa la opción --no-merges.

filtros multicriterio

Cuando se limita la salida del comando log usando múltiples criterios, por defecto se obtendrán todas aquellas confirmaciones que cumplan con alguno de los criterios, como si de la operación lógica OR se tratara. Si deseas por el contrario aplicar la operación lógica AND debes usar la opción --all-match.

filtrando por contenido de directorio

Una opción verdaderamente útil para filtrar la salida de git log es especificar una ruta. Si especificas la ruta de un directorio o archivo, puedes limitar la salida a aquellas confirmaciones que introdujeron un cambio en dichos archivos. Ésta debe ser siempre la última opción, y suele ir precedida de dos guiones (--) para separar la ruta del resto de opciones.

git log --since="2008-10-01" --no-merges -- t/

Mostrar las diferencias introducidas en cada confirmación

Podemos pedir que se muestran las diferencias introducidas en cada confirmacción utilizando la opción -p

git log -p
commit ca82a6dff817ec66f44342007202690a93763949
Author: Scott Chacon 
Date:   Mon Mar 17 21:52:11 2008 -0700

    changed the version number

diff --git a/Rakefile b/Rakefile
index a874b73..8f94139 100644
--- a/Rakefile
+++ b/Rakefile
@@ -5,7 +5,7 @@ require 'rake/gempackagetask'
 spec = Gem::Specification.new do |s|
-    s.version   =   "0.1.0"
+    s.version   =   "0.1.1"
     s.author    =   "Scott Chacon"

Esta opción muestra la misma información, pero añadiendo tras cada entrada las diferencias que le corresponden. Esto resulta muy útil para revisiones de código, o para visualizar rápidamente lo que ha pasado en las confirmaciones enviadas por un colaborador.

cambios a nivel de palabra

A veces es más fácil revisar cambios a nivel de palabra que a nivel de línea. Git dispone de la opción --word-diff, que se puede añadir al comando git log -p para obtener las diferencias por palabras en lugar de las diferencias línea por línea. Formatear las diferencias a nivel de palabra es bastante inusual cuando se aplica a código fuente, pero resulta muy práctico cuando se aplica a grandes archivos de texto, como libros o tu propia tesis. He aquí un ejemplo:

git log -U1 -p --word-diff

Como se puede ver, no aparecen líneas añadidas o eliminadas en la salida como en las diferencias normales. Se puede ver la palabra añadida encerrada en {+ +} y la eliminada en [- -]. Puede que se quiera reducir las usuales tres líneas de contexto en las diferencias a sólo una línea puesto que el contexto es ahora de palabras, no de líneas. Se puede hacer esto con -U1, como hicimos en el ejemplo de arriba.

Estadísticas de cada confirmación

Para ver algunas estadísticas de cada confirmación, usa la opcion --stat, que imprime tras cada confirmación una lista de archivos modificados, indicando cuántos han sido modificados y cuántas líneas han sido añadidas y eliminadas para cada uno de ellos, y un resumen de toda esta información.

git log --stat

resumen de las estadísticas

La opción --shortstat muestra solamente la línea de resumen de la opción --stat.

git log --shortstat

Lista de archivos modificados

Es posible ver exactamente que archivos se han modificado en cada confirmación, agregando al comando git log una de las siguientes opciones.

Si tu objetivo es mostrar solamente la lista de todos los archivos afectados en la confirmación, es decir, todos aquellos archivos añadidos, modificados o afectados, basta con usar la opción --name-only:

git log --name-only

Si además deseamos que la lista indique explisitamente si los archivos fueron añadidos, modificados o eliminados, deberás usar la opción --name-status.

git log --name-status

Modificando el formato de salida

La opción --pretty modifica el formato de salida. Tienes unos cuantos estilos disponibles. La opción oneline imprime cada confirmación en una única línea, lo que puede resultar útil si estás analizando gran cantidad de confirmaciones. Otras opciones son short, full y fuller, que muestran la salida en un formato parecido, pero añadiendo menos o más información, respectivamente:

git log --pretty=oneline

opciones rápidas de formato

--abbrev-commit

Muestra solamente los primeros caracteres de la suma SHA-1, en vez de los 40 caracteres de que se compone.

--relative-date

Muestra la fecha en formato relativo (por ejemplo, “2 weeks ago” (“hace 2 semanas”)) en lugar del formato completo.

opción format

La opción más interesante es format, que te permite especificar tu propio formato. Esto resulta especialmente útil si estás generando una salida para que sea analizada por otro programa:

git log --pretty=format:"%h - %an, %ar : %s"

a siguiente lista algunas de las opciones más útiles aceptadas por format.

Opción Descripción de la salida Opción Descripción de la salida
%H Hash de la confirmación %h Hash de la confirmación abreviado
%T Hash del árbol %t Hash del árbol abreviado
%P Hashes de las confirmaciones padre %p Hashes de las confirmaciones padre abreviados
%an Nombre del autor %ae Dirección de correo del autor
%ad Fecha de autoría (el formato respeta la opción -–date) %ar Fecha de autoría, relativa
%cn Nombre del confirmador %ce Dirección de correo del confirmador
%cd Fecha de confirmación %cr Fecha de confirmación, relativa
%s Asunto

opción graph

Las opciones oneline y format son especialmente útiles combinadas con otra opción llamada --graph. Ésta añade un pequeño gráfico ASCII mostrando tu histórico de ramificaciones y uniones, como podemos ver en el siguiente ejemplo:

git log --pretty=format:"%h %s" --graph
